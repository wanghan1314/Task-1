1.背景介绍

为什么要Cookie和Session？

因为http是一种不状态性的协议；

Http协议是无状态的协议，一旦数据库交换完毕，客户端与服务器的链接会关闭，再次交换数据，需要建立新的链接。（服务器无法从连接上跟踪会话，对于上一个会话遗留的信息，无法判断是属于哪个请求的会话）

1.背景介绍

什么是有状态？

需要弥补HTTP协议无状态的不足；基本在Session出现前，都用Cookie来跟踪会话

Cookie和Session存在的意义就是为了保存Http客户端/服务器的部分有用的状态。

1.背景介绍

Cookies：保存在客户端浏览器中，相当于确认客户身份的“通行证”；

Session：保存在服务器上，相当于客户档案，客户明细表

Cookie部分：

Cookie生存期和存放的位置：

生存期：

-1:如果Cookie的生存期是整个会话期间，那么Cookie保存在内存中，浏览器关闭的时候就会自动清除这个Cookie。

-2:如果Coolie的生存期设定了一个长的值，那么浏览器关闭Cookie也不会被清除，Cookie会保存在硬盘中。下次访问的时候，就会在请求头携带这个Cookie。

-3:存放位置：windows: [系统盘]:\Documents and Settings[用户名]\Cookies。

.2知识剖析

Cookie部分：

Cookie的设置过程

Cookie的设置分成以下4步

HTTP Request

HTTP Response + Set-Cookie

HTTP Request + Cookie

HTTP Response

虽然在二次请求的时候，用户可能禁止Cookie的使用，但是这种情况的出现很少。

.2知识剖析:Cookie部分：

Cookie的设置和获得

Cookie 封装成javax.servlet.http.Cookie类。

服务器通过操作Cookie类对象，对客户端Cookie进行操作。

Request.getCookie() //获取客户端提交的所有Cookie， 以Cookie[]数组形式返回

Response.addCookie(Cookie cookie) //向客户端设置Cookie.

每一Cookie对象以 key-value的属性对形式，保存用户形态；

一个request和response可以同时包含和使用多个cookie（有上限）；需要的时候对该cookie进行遍历； Cookie类，位于包javax.servlet.http.*下面；

Cookie部分：属性

4.1:name 和 value

Name 和 Value是在reponse Cookie之前就设置的。

name和value是成对存在的。

通过name和value的设置，在下次访问的时候，通过name和value就能判断Cookie是哪个客户端，进而响应相应的资源；

.2知识剖析

Cookie部分：属性

4.2:maxAge:

表示Cookie的有效期。

=-1，表示关闭就失效，默认就是-1；

=0，表示删除cookie，cookie没有删除cookie的机制，只能设置即使失效，删除它；

>0, 表示设置长久的cookie,这个时候会进行倒数，倒数到0就会删除。

.2知识剖析

Cookie部分：属性

4.3:secure

HTTP. 无状态且不安全；不加密有可能被截获；

HTTPS和SSL是安全协议；

将Cookie的secure属性设置为true，就是指允许在HTTPS和SSL等安全协议中传输此类Cookie。

Cookie.setSecure(true);

__这只是在传输协议上的加密；提高安全性，需要对内容进行加密，解密；

.2知识剖析

Cookie部分：属性

4.4:path

Path属性需要使用符号“/”结尾

只允许/xxxweb/下的程序使用cookie（决定允许访问Cookie的路径【ContexPath】）

设置成"/"允许所有路径。

页面只能获取属于它Path的cookie: /session/test/a.jsp不能获得/session/abc的Cookie。

.2知识剖析:Cookie部分：属性

4.5.1:Domain：访问域

*隐私机制决定不可跨域名

*一般情况下，同一个一级域名下的两个二级域名如

123.com；Images.123.com

也不能交互使用Cookie，因为域名并不严格完全相同；

若想所有二级域名都可以使用该Cookie，需要设置Cookie的Domain参数；

Cookie cookie = new Cookie("time","2008080");

Cookie.setDomain(".123.com");//设置域名，这样123.com的所有子域名可以使用该cookie

.2知识剖析:Session部分：

0:Session的解释：

Session是服务器端使用的一种记录客户端状态的机制。

比Cookie简单一些，也相应的增加了服务器的存储压力；

Q：如何根据Value-key从Session中获取匹配的内容；

A：创建session ————在服务器端程序运行的过程中创建；(同时为该Session生成唯一的Session ID)

.2知识剖析

2.1:Session的生命周期

1：Session在内存中；

优点：提高存取速度；弊端：大量访问会导致内存溢出

要求：Session里面的内容尽量精简。

2:Session第一次访问的时候创建；

未生成Session可用：Request.getSession(true）强制生成；

.2知识剖析

访问：JSP，Servlet等程序时候才会创建Session；访问静态资源，不会创建Session。

3:Session更新，更新到Session的最后访问时间，无论是否读写Session 都认为该用户的Session活跃了一次

.2知识剖析:Session部分：

2.2:Session的有效期

为了防止资源浪费，需要设置一个有效期。

.2知识剖析

2.3Cookie和Session的区别：

1cookie数据存放在客户的浏览器上，session存放在服务器上。

2cookie不太安全，能分析存放在本地的cookie，进行cookie欺骗；安全的话，应该使用session；

3session会在一定时间内保存在服务器，当访问过多，会比较占用服务器的性能，如果要减轻服务器性能的压力，可以使用cookie；

.2知识剖析

2.3Cookie和Session的区别：

4单个cookie在客户端的限制的3K，一个站点在客户点存放的cookie不能超过3K

session机制会生成session的id，该id作为识别标志存放在客户端的cookie容器中，客户端每次请求，会把该id放到http的请求的头部，发送给服务器。

对于jsp和php编写的网站，完全禁用cookie的时候，session会不能正常使用。
